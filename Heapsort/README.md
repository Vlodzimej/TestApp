## Выводы

- Сортировка пузырьком является самым медленным алгоритмом из реализованных и уже следующая сортировка выбором работает в 5 раз быстрее.

- Однако, если при сортировке **10.000** элементов **Selection Sort** показывает приемлемое время выполнение за **1 секунду**, то уже при **100.000** чисел
  алгорит завершает работу примерно за **10 секунд**, то есть время выполнение возрастает, как и в предыдущем случае, прямо пропорцинально увеличению
  количества элементов сортируемого массива. Обработка отсортированного массива в обоих случаях никак не сказывается на производительности.

- Алгоритм **Insertion Sort** выполняется немного быстрее предыдущего и показывает хороший результат при обработке отсортированного массива,
  при этом его сложность близка к **O(n)**, но с реверсированным массивом имеет отрицательную эффективность.

- Сортировка Шелла работает в несколько раз быстрее остальных алгоритмов. С увеличением количества шагов возрастает и производительность, при этом
  его сложность сохраняется. При работе с частично отсортированным массивам сложность может стремиться к **O(n log<sub>n</sub>)**.
  Это первый алгоритм в данной работе, который показал приемлемый результат при обработке массива из **10<sup>7</sup>** элементов, выполнив её за **13 минут**.

- Пирамидальная сортировка доказывает свою эффективность по всем тестам, хотя и не является наиболее быстрым алгоритмом, как например **Quick Sort** и **Merge Sort**.
